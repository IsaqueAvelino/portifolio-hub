# PortfolioHUB - Repositório de Versionamento Profissional

Este repositório foi desenvolvido como entrega intermediária para o desafio acadêmico PortfolioHUB. O objetivo principal é consolidar projetos acadêmicos e pessoais, aplicando as melhores práticas de versionamento com Git/GitHub e estabelecendo uma ponte robusta com o mercado de trabalho por meio da integração com plataformas profissionais.

## 📁 Estrutura de Diretórios do Repositório
Para cumprir as seções mínimas exigidas pelo planejamento e organização do projeto, a árvore de diretórios foi estruturada da seguinte forma:

- `/projetos-academicos`: Pasta dedicada ao armazenamento de projetos, códigos, slides e documentos desenvolvidos no âmbito universitário.
- `/projetos-pessoais`: Pasta destinada a estudos autônomos, testes de código e ferramentas de desenvolvimento pessoal.
- `index.html`: Arquivo principal contendo a estrutura da página web inicial (Versão 1.0 do projeto).

## 🛠️ Tecnologias e Ferramentas Utilizadas
- **Git:** Utilizado para o controle de versão local e registro do histórico do código.
- **GitHub:** Plataforma de hospedagem na nuvem utilizada para o compartilhamento seguro do código.
- **GitHub Pages:** Ferramenta responsável pelo deploy automatizado e publicação da página estática do portfólio.
- **Markdown:** Linguagem de marcação empregada para a construção desta documentação técnica oficial.

## 🚀 Histórico de Versionamento (Boas Práticas)
1. **Inicialização:** `git init` executado localmente.
2. **Registro de Identidade:** Configuração local de credenciais (`user.name` e `user.email`).
3. **Versão 1.0:** Criação da página web inicial (`index.html`) e execução do primeiro commit estrutural (`git commit -m "versao 1.0"`).
4. **Deploy Remoto:** Vinculação com o repositório remoto do GitHub e envio dos arquivos via comando `git push`.
